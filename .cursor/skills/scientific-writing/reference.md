# Scientific writing: reference

Read this file when the user wants deeper guidance, IMRAD detail, or revision passes.

## Paragraph pattern (PEEL variant)

1. **Point**: One claim for the paragraph.
2. **Evidence**: Data, prior work, or definition that supports it.
3. **Explanation**: Why the evidence supports the point (not repetition).
4. **Link**: Transition to the next paragraph or limitation.

## Abstract checklist

- [ ] States the problem or gap in one sentence
- [ ] States approach or design without unnecessary jargon
- [ ] States main quantitative or qualitative outcome if applicable
- [ ] States implication or contribution (not hype)

## Related work grouping

Group papers by **problem framing or method family**, not alphabetically. For each group: what they did, how your work differs, one fair limitation of their setup if comparing.

## Methods: minimum credible detail

- Population or materials: what, from where, inclusion rules if relevant
- Design: independent variables, conditions, controls
- Procedure: order of steps a reader could replicate
- Analysis: model or test, assumptions, preprocessing, software only if venue expects it

## Discussion balance

Aim for roughly: interpretation of main findings, comparison to prior work, limitations (specific), and future work or generalization (bounded).

## Revision pass

1. Cut filler and duplicate sentences.
2. Align every strong claim with nearby support or hedge.
3. Unify terminology.
4. Check that limitations are present if the text makes empirical claims.
